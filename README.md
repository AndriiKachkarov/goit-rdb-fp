# Реляційні бази даних — Фінальний проєкт (Pandemic Data Analysis)

Цей репозиторій містить фінальну роботу, присвячену аналізу глобальних пандемічних даних. Проєкт охоплює повний цикл роботи з даними: створення схеми, нормалізацію до 3НФ, агрегатний аналіз та розробку користувацьких функцій.

## 📂 Структура проєкту

- **p1**: Створення схеми `pandemic` та первинний імпорт даних.
- **p2**: Процес нормалізації даних. Початкова таблиця розділена на дві: `entities` (довідник країн) та `cases_data` (статистика захворювань). Це дозволило усунути надмірність даних.
- **p3**: Розрахунок статистичних показників (`AVG`, `MIN`, `MAX`, `SUM`) для захворюваності на сказ.
- **p4**: Маніпуляції з датами: перетворення року на об'єкт `DATE` та розрахунок різниці з поточною датою.
- **p5**: Створення SQL-функції `GetYearDiff` для автоматизації розрахунку часових інтервалів.

## 🛠 Ключові етапи виконання

### 1. Нормалізація (3rd Normal Form)

Для досягнення третьої нормальної форми було проведено декомпозицію:

- Створено таблицю-довідник `entities`, де кожна країна та її код зберігаються в одному екземплярі.
- Створено таблицю фактів `cases_data`, яка посилається на `entities` через `entity_id`.

**Результат:** Обсяг бази даних оптимізовано, усунуто ризик аномалій при оновленні назв країн.

### 2. Аналіз захворюваності на сказ

- За допомогою агрегатних функцій проведено аналіз атрибута `Number_rabies`.
- Використано фільтрацію `WHERE ... <> ''`, щоб виключити порожні значення та отримати точне середнє арифметичне.
- Отримано ТОП-10 країн за середнім показником захворюваності.

### 3. Робота з датами та функціями

- Реалізовано логіку перетворення цілочисельних значень року у повноцінний часовий формат.
- Функція `GetYearDiff` інкапсулює складну логіку `STR_TO_DATE` та `TIMESTAMPDIFF`.
- Це дозволяє динамічно отримувати "вік" даних відносно поточної дати (`CURDATE()`).

## 📊 Результати

Всі запити перевірені в MySQL Workbench та готові до використання. Кількість імпортованих рядків зафіксована запитом `SELECT COUNT(*)`, що підтверджує цілісність даних після завантаження.
